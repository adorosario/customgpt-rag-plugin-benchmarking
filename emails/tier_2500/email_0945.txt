Hey everyone,

Attached is the postmortem for the WebhookDispatcher outage that occurred on Thursday, January 4. The
incident lasted approximately 45 minutes and affected 93 for a subset of users.

Root cause: a configuration change in the caching layer deployment introduced a race
condition under high load. The issue manifested when concurrent requests exceeded
389ms, causing connection pool exhaustion.

Timeline: the alert fired at 14:23 UTC, Daniel Park acknowledged within 3 minutes,
and the fix was deployed by 15:08 UTC. Detection was fast thanks to the new alerting
rules, but resolution took longer than ideal because the runbook for this service
was outdated.

Action items: update the WebhookDispatcher runbook, add a circuit breaker for the
caching layer connection pool, and implement load shedding when concurrency exceeds
389ms. I've created tickets for all three.

Please review and add any corrections by end of week.
