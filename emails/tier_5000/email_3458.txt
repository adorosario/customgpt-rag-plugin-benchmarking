Attached is the postmortem for the PaymentGateway outage that occurred on Wednesday, August 25. The
incident lasted approximately 45 minutes and affected 73 for a subset of users.

Root cause: a configuration change in the event bus deployment introduced a race
condition under high load. The issue manifested when concurrent requests exceeded
330ms, causing connection pool exhaustion.

Timeline: the alert fired at 14:23 UTC, Liam Foster acknowledged within 3 minutes,
and the fix was deployed by 15:08 UTC. Detection was fast thanks to the new alerting
rules, but resolution took longer than ideal because the runbook for this service
was outdated.

Action items: update the PaymentGateway runbook, add a circuit breaker for the
event bus connection pool, and implement load shedding when concurrency exceeds
330ms. I've created tickets for all three.

Please review and add any corrections by end of week.
